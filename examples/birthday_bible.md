# 事例: 鏡リョウジ誕生日バイブル

## 背景

- **対象**: 鏡リョウジ『誕生日バイブル』(365日分 × 24カラムの構造化データ)
- **入力**: iXScan で書籍を全ページスキャンしたPDF(765ページ, 135MB)
- **出力**: 1枚のXLSXシートに 366行 × 24カラム

### 1枚のページの構造

見開き2ページで1日分。各日で以下を抽出:

| カラム | 抽出元 | 型 |
|---|---|---|
| 誕生日 | 左ページ上部 | date |
| キーワード / エレメント / 活動星座 / デーク惑星 / シークレット星座 / ルーン | 左サイドバー | string |
| タロット / クリスタル / 色 / アイテム | 左サイドバー | JSON array |
| リード / リード本文 / 愛の形 / 才能テーマ | 左右ページ本文 | string |
| 長所 / 短所 / 著名人 | 右カラム | JSON array |
| 守護聖人 | 右カラム | string |
| 恋人 / 友人 / 手本 / 助け / 縁 | 右下相性リスト | JSON array of "M月D日" |

## パイプライン

```
PDF → PNG(300dpi) → 左右ページクロップ → 5ブロッククロップ → PaddleOCR → 正規表現パース → XLSX
```

### 工夫点

1. **ページ→日付マッピングの自動検出**
   書籍には章扉(星座)が挿入されるため、通常の `2ページ=1日` 計算が崩れる。
   各ページ左上をOCRして `N月D日` を検出 → 自動マッピング生成。

2. **5ブロッククロップ**
   ページ全体ではなく、レイアウトに沿って左サイドバー/本文/長所短所/著名人/相性を別々にOCRすることで、構造抽出の正規表現がシンプルに。

3. **PaddleOCR mobile + 並列**
   `PP-OCRv5_mobile` モデル + `ProcessPoolExecutor(workers=4-6)` で 1830クロップを約60分で処理。

4. **段階的パーサ**
   サイドバーは `【キーワード】` などの見出し正規表現でセクション分割 → 誤字許容の正規表現(例: `【\s*キ[ーー]?ワ[ーー]?[ドト]\s*[】\]]?`)。

5. **配列型への変換**
   - 長所/短所は句点「。」で分割 → 箇条書き配列
   - クリスタル/色/アイテムは読点で分割
   - 相性リストは「M月D日」文字列配列へ正規化
   - 著名人は `名前(職業)` 形式の配列へ

## 結果

- 365/366日 のサイドバー・本文・相性リストが自動で抽出された
- 残る誤字は OCR由来の数文字/日 程度(「夕→タ」「一→ー」「じる→しる」など辞書で吸収可能)
- 作業時間:
  - PDF→PNG化: 15分
  - 日付検出: 5分
  - クロップ: 2分
  - OCR(並列): 60分
  - XLSX書き込み: 秒

## 学び

- **Tesseract日本語は書籍スキャンに弱い**。PP-OCRv5 に切り替えるだけで誤字が1/5以下に。
- **Apple Silicon には mobile モデル + 並列** が圧倒的にコスパが良い(server版は数倍時間がかかる)。
- **ページマッピングは自動検出化しておく**と章扉やスキャン漏れにも対応できる。
